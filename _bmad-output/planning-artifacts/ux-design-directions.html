New v2 Elements in This Layout

  • AI Sparkle Icon (toolbar, far right): Single entry point for AI features. Pulsing border draws attention on first visit.
  • Auth Button (header, far left): Minimal "Login" button that becomes an avatar after login. No navigation bar.
  • Everything else: Identical to v1 layout. Same toolbar groups, same view toggles, same color panel access.

Keyboard shortcut: Ctrl+K / Cmd+K. RTL-native. Contextual AI actions.

Design Decisions

  • RTL-native: Input placeholder, action labels, and descriptions all flow right-to-left.
  • Contextual actions: The palette shows actions relevant to document content, not a generic menu.
  • Keyboard-first: Arrow keys navigate, Enter selects, Esc closes. Mouse works too.
  • No chatbot: This is an action menu, not a conversation. Select an action, get a result.
  • Gate check: Anonymous users see a "Register free" prompt inside the palette, not a blocking modal.

Design Decisions

  • Single button, far left: Auth lives at the end of the header actions, after the separator. Visually separated from tool actions.
  • Anonymous: Subtle outlined button "Login / Register". Not prominent. Tool works without it.
  • Logged in: Initials avatar replaces the button. Clicking opens a dropdown: account info, tier status, logout.
  • Paid badge: Small gold dot on the avatar indicates paid tier. Subtle, not flashy.
  • No navbar: There is no "Home / Pricing / Features" navigation. The editor IS the product.

Design Decisions

  • Recommendation: Use both B and C together. Show limit inside the command palette (Variant B) when user opens AI, and a toast (Variant C) after a failed attempt. Variant A (toolbar banner) is optional and more intrusive.
  • Never modal: No popup that blocks the screen. No "Subscribe now!" overlay.
  • Dismissible: Banner has an X button. Toast auto-dismisses. User is never stuck.
  • Informative: Shows usage (3/3) so user understands the boundary. Not "you can't do this" but "you've used your allocation."
  • Upgrade link, not a wall: The upgrade button is a gentle CTA, not a demand.

4. Presentation Mode

Full-screen, distraction-free preview. All chrome fades away.
Controls appear on hover. Escape returns to editor.

תכנון ספרינט הבא

הצוות דן בעדיפויות לספרינט הקרוב. המשימות העיקריות כוללות:

  • שיפור ביצועי הייצוא ל-PDF
  • תמיכה ב-RTL אוטומטית
  • אינטגרציה עם Claude API
הערה חשובה: יש לסיים את המיגרציה לפני סוף החודש.

החלטות

1. מועד השקה: 15 באפריל
2. תקציב: אושר
3. צוות: 3 מפתחים

Design Decisions

  • Full viewport: No header, no footer, no sidebar. The rendered content fills the screen.
  • Larger text: Font sizes increase ~10-15% for readability on projected screens.
  • Hover controls (top-left): Exit (X), theme toggle, and export. Fade in on mouse move, fade out after 3s idle.
  • Escape to exit: Press Esc to return to the normal editor view.
  • Color theme preserved: Whatever preset the user selected carries into presentation mode.
  • Entry point: New "Presentation" option in the view toggle group, or a dedicated button in the header.

Design Decisions

  • Phase 1 is dead simple: Filename (auto-filled from first heading) + export button. That's it.
  • Phase 3 adds progressive disclosure: Paper size, margins, cover page, page numbers. Still fits in one dialog.
  • Progress toast: Sonner toast with spinner during PDF generation. Updates to checkmark on completion.
  • Smart defaults: A4, normal margins, no cover page, page numbers on. Most users never change these.
  • RTL dialog: All labels, inputs, and buttons flow right-to-left natively.
